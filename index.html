Day 1 Logic & Truth Tables — Deep Refresh Review + Extend ▾

📋 What You Need Before Starting

You said you already know Chapter 1 parts 1–3. This day is about plugging the gaps: quantifiers, nested quantifiers, and the logical equivalence proof techniques that appear on harder exams. Do NOT skip this because you think you know it. The exam goes harder than the midterm.

Core Connectives — Must Know

Exam Traps — Logic

  • p → q with p = F: The whole implication is TRUE. Always. This is called vacuous truth.
  • Converse vs Contrapositive: Contrapositive ¬q → ¬p is equivalent to p → q. The converse q → p is NOT equivalent.
  • p ↔ q is true in exactly 2 rows (TT and FF), not 3.
  • XOR vs OR: p ∨ q includes the case where both are true. p ⊕ q does not.
  • n variables → 2ⁿ rows in a truth table. Never miscound.

The Implication Family

Original: p → q Converse: q → p ← NOT equivalent to original Inverse: ¬p → ¬q ← NOT equivalent to original Contrapositive: ¬q → ¬p ← EQUIVALENT to original ✓ // Key rewrite (memorize this!) p → q ≡ ¬p ∨ q

Logical Equivalences — The Non-Negotiable List

De Morgan's: ¬(p ∧ q) ≡ ¬p ∨ ¬q ¬(p ∨ q) ≡ ¬p ∧ ¬q Identity: p ∧ T ≡ p | p ∨ F ≡ p Domination: p ∨ T ≡ T | p ∧ F ≡ F Idempotent: p ∨ p ≡ p | p ∧ p ≡ p Double Neg: ¬(¬p) ≡ p Absorption: p ∨ (p ∧ q) ≡ p Implication: p → q ≡ ¬p ∨ q Biconditional: p ↔ q ≡ (p → q) ∧ (q → p)

Quantifiers — What the Exam Will Test You On

Universal (∀) vs Existential (∃)

  • ∀x P(x): P(x) is true for every x in domain. One counterexample makes it false.
  • ∃x P(x): P(x) is true for at least one x. One example makes it true.
  • ¬∀x P(x) ≡ ∃x ¬P(x) — De Morgan's for quantifiers
  • ¬∃x P(x) ≡ ∀x ¬P(x) — flip quantifier, negate predicate

Universal vs Existential — Translation Rule

  • "All S are P" → ∀x (S(x) → P(x))   NOT ∀x(S(x) ∧ P(x))
  • "Some S are P" → ∃x (S(x) ∧ P(x)) NOT ∃x(S(x) → P(x))
Classic error: Students mix → and ∧ between universal and existential. Universal uses →. Existential uses ∧.

Nested Quantifiers — Order Matters

StatementWhen TrueWhen False
∀x ∀y P(x,y)True for every pairEven one pair fails → false
∀x ∃y P(x,y)For each x, some y worksSome x has no working y
∃x ∀y P(x,y)One x works for all yNo single x works for all y
∃x ∃y P(x,y)Some pair (x,y) worksNo pair works at all
Critical: ∀x ∃y P(x,y) is NOT the same as ∃y ∀x P(x,y). The second requires ONE y that works for every x simultaneously.

You need to be completely solid on Day 1 — specifically p → q truth table and logical equivalences. Rules of inference are just structured applications of those equivalences.

Proof Methods — Know When to Use Each

Direct Proof

1
Assume p is true

State your assumption clearly

2
Use definitions + logic

Write out definitions (e.g. n is odd → n=2k+1)

3
Show q must be true

Conclude the desired property

Example: Prove "if n is odd then n² is odd" Assume n is odd → n = 2k+1 for some integer k n² = (2k+1)² = 4k²+4k+1 = 2(2k²+2k)+1 → n² is odd ✓

Proof by Contrapositive

Instead of proving p → q, prove ¬q → ¬p (equivalent). Use when a direct proof gets stuck.

Example: Prove "if n² is odd, then n is odd" (Direct is hard — try contrapositive) Assume n is even → n = 2k n² = (2k)² = 4k² = 2(2k²) → n² is even ∴ By contrapositive, n² odd → n odd ✓

Proof by Contradiction

1
Assume ¬p (the opposite)

Suppose what you want to prove is false

2
Derive a contradiction

Show you reach something impossible (A ∧ ¬A)

3
Conclude p must be true

Since ¬p leads to impossibility

Classic example from slides: Proving √2 is irrational. Assume √2 = m/n in lowest terms. Show both m and n must be even — contradiction with "lowest terms".

Vacuous & Trivial Proofs

  • Vacuous: If p is always false, then p→q is always true. (You prove ¬p.)
  • Trivial: If q is always true, then p→q is always true. (You prove q.)
  • Both are real techniques, not tricks. Appear in induction base cases.
Biconditional proofs: To prove p↔q you must prove BOTH p→q AND q→p. Half credit if you only do one direction.

What the Exam Expects — Step Format

Step 1: ¬p ∧ q — Premise Step 2: ¬p — Simplification (Step 1) Step 3: q — Simplification (Step 1) Step 4: p → r — Premise Step 5: ¬r — Modus Tollens (Step 2, Step 4) Step 6: q ∧ ¬r — Conjunction (Step 3, Step 5)
Every step needs a justification. Writing the result without naming the rule used will cost you marks.
Day 3 Sets — All Operations and Identities Fresh Topic ▾

📋 Prerequisites

Set identities mirror logical equivalences exactly. If you know De Morgan's for logic, you already know De Morgan's for sets. The connection is: ∪ corresponds to ∨, ∩ corresponds to ∧, complement corresponds to ¬.

Core Definitions

What You Must Know Cold

  • A ⊆ B: every element of A is in B. ∅ ⊆ every set. Every set ⊆ itself.
  • A = B: A ⊆ B AND B ⊆ A (both ways)
  • Proper subset A ⊂ B: A ⊆ B but A ≠ B
  • Power set P(A): all subsets of A. |A|=n → |P(A)| = 2ⁿ
  • Cardinality |A|: number of distinct elements
  • Cartesian product A × B: all ordered pairs (a,b) with a∈A, b∈B

The Empty Set Traps

  • ∅ ≠ {∅}. The empty set has 0 elements. {∅} has 1 element.
  • |P(∅)| = 2⁰ = 1. P(∅) = {∅}.
  • ∅ is a subset of every set — including itself.
  • A set with n elements has 2ⁿ subsets. Count ∅ and A itself.
Quick check: subsets of {1,2}
∅, {1}, {2}, {1,2} — that's 4 = 2² subsets. ✓

Set Operations

OperationSymbolDefinitionVenn
UnionA ∪ Bx∈A or x∈B (or both)both circles filled
IntersectionA ∩ Bx∈A and x∈Boverlap only
ComplementĀx∈U but x∉Aeverything outside A
DifferenceA − Bx∈A and x∉B = A ∩ B̄A minus overlap
Sym. DifferenceA ⊕ B(A∪B) − (A∩B)both circles, no overlap

Inclusion-Exclusion (Cardinality)

|A ∪ B| = |A| + |B| − |A ∩ B| // Always subtract the intersection — you counted it twice! // Example: A={1,2,3,4,5}, B={4,5,6,7,8} // |A∪B| = 5 + 5 − 2 = 8 ✓

Set Identities (Mirror of Logic Laws)

De Morgan's: A ∪ B ̄ = Ā ∩ B̄ and A ∩ B ̄ = Ā ∪ B̄ Distributive: A ∪ (B ∩ C) = (A ∪ B) ∩ (A ∪ C) Absorption: A ∪ (A ∩ B) = A Complement: A ∪ Ā = U and A ∩ Ā = ∅ Identity: A ∪ ∅ = A and A ∩ U = A

How to Prove a Set Identity (3 Methods)

1
Subset method

Show A⊆B then B⊆A. Takes longest but always works.

2
Set-builder + propositional logic

Translate using x∈A↔P(x), then manipulate with logical equivalences.

3
Membership table

Like a truth table. 1 = element is in the set. Same columns = identical sets.

Day 4 Sequences + Summations — The Calculation Day Hard — Needs Drill ▾

📋 Prerequisites

Basic algebra. You need to be comfortable identifying what is a constant vs a variable inside a summation (the index is the only variable). This day requires you to practice problems by hand — reading is not enough.

Sequences

Types of Sequences

  • Arithmetic: aₙ = a + (n−1)d. Add d each time. Ex: 1,3,5,7 (d=2)
  • Geometric: aₙ = arⁿ. Multiply by r each time. Ex: 2,6,18,54 (r=3)
  • Fibonacci: f₀=0, f₁=1, fₙ = fₙ₋₁ + fₙ₋₂

Fibonacci — Compute Fast

f₀=0, f₁=1, f₂=1, f₃=2, f₄=3, f₅=5, f₆=8, f₇=13, f₈=21, f₉=34, f₁₀=55

Memorize the first 10 terms. Exam questions often ask you to compute fₙ from scratch.

Summation — The Central Skill

// The "what is a variable?" rule — MOST IMPORTANT // Only the INDEX LETTER is a variable inside a summation. // Everything else is a constant, no matter what it looks like. ∑(i=1 to n) 42j ← i is index, j is CONSTANT → answer: 42j·n ∑(Bob=1 to n) Jeff ← Bob is index, Jeff is CONSTANT → answer: Jeff·n

Closed-Form Formulas — Must Memorize All of These

Sum of 1s: ∑(i=1 to n) 1 = n Sum of i: ∑(i=1 to n) i = n(n+1)/2 Sum of i²: ∑(i=1 to n) i² = n(n+1)(2n+1)/6 Sum of i³: ∑(i=1 to n) i³ = [n(n+1)/2]² Constant k: ∑(i=1 to n) k = k·n Split sums: ∑(f(i)+g(i)) = ∑f(i) + ∑g(i) Change limits: ∑(i=m to n) f(i) = ∑(i=1 to n) f(i) − ∑(i=1 to m−1) f(i)
The Limit-Shift Rule is the Most Tested
When the lower limit is NOT 1, you must subtract: ∑(i=m to n) = ∑(i=1 to n) − ∑(i=1 to m−1). This is how your professor solved Problems 4 and 5 in the slides. Practice this until it is automatic.

Nested Summations — Work from Right to Left

∑(i=1 to n) ∑(j=1 to m) c ← start with inner (j), c is constant = ∑(i=1 to n) c·m ← c·m is constant w.r.t. i = c·m·n ∑(i=1 to n) ∑(j=1 to i) c ← inner gives c·i (i appears!) = ∑(i=1 to n) c·i ← now i IS a variable = c · n(n+1)/2
Day 5 Functions — Injections, Surjections, Bijections New Topic ▾

📋 Prerequisites

Sets (Day 3). Functions are defined as a subset of A×B. You also need to know what domain, codomain, and range mean precisely — these are different things and the exam will test that distinction.

Core Vocabulary — Every Term Will Appear on the Exam

TermDefinitionExample
DomainThe set A that f maps FROMf: ℤ → ℤ, domain = ℤ
CodomainThe set B that f maps TOf(x)=x², codomain = ℤ
Range / ImageActual output values (subset of codomain)f(x)=x², range = {0,1,4,9,...}
PreimageWhich input maps to a given outputPreimage of 4 under x² is {2,−2}
Range ≠ Codomain. Codomain is what you declare; range is what you actually hit. f: ℤ→ℤ with f(x)=x² has codomain ℤ but range {0,1,4,9,16,...} — not all integers are squares.

The Three Types of Functions

Injection (One-to-One)

Rule: f(a) = f(b) ⟹ a = b. Different inputs give different outputs.

To PROVE injection: Assume f(a) = f(b), show a = b algebraically. To DISPROVE: Find two inputs a ≠ b with f(a) = f(b). Example: f(x) = 2x+1 IS injective Example: f(x) = x² is NOT injective (f(2)=f(−2)=4, but 2≠−2)

Surjection (Onto)

Rule: For every b∈B, ∃a∈A such that f(a)=b. Every element of codomain is hit.

To PROVE surjection: Take an arbitrary b in codomain. Solve for a in terms of b. Show a is in the domain. Example: f: ℤ→ℤ, f(x)=x+1 IS surjective Example: f: ℤ→ℤ, f(x)=x² is NOT surjective (no integer squares to −1)

Bijection = Injective + Surjective

A bijection is a perfect one-to-one correspondence. Only bijections have inverses. If f is not a bijection, f⁻¹ does not exist.

Inverse Functions

f must be a bijection for f⁻¹ to exist. f⁻¹(y) = x iff f(x) = y Example: f(x) = x + 1 Inverse: f⁻¹(y) = y − 1 (Every output y maps back to exactly one input x = y−1)

Function Composition

(f ∘ g)(x) = f(g(x)) ← apply g first, then f Example: f(x) = 2x+3, g(x) = 3x+2 (f ∘ g)(x) = f(3x+2) = 2(3x+2)+3 = 6x+7 (g ∘ f)(x) = g(2x+3) = 3(2x+3)+2 = 6x+11 // NOTE: f∘g ≠ g∘f in general!

Floor and Ceiling

⌊x⌋ = largest integer ≤ x (floor) ⌈x⌉ = smallest integer ≥ x (ceiling) ⌊3.7⌋ = 3 ⌈3.7⌉ = 4 ⌊−2.3⌋ = −3 ⌈−2.3⌉ = −2 ← negative numbers trip people up!
Day 6 Relations — Properties, Matrices, Digraphs, Equivalence High Volume ▾

📋 Prerequisites

Sets (Day 3). A relation on A is a subset of A×A. Functions are a special case of relations. You need to be comfortable reading and writing ordered pairs.

The Four Properties — Know These Definitions Exactly

PropertyDefinitionMatrix TestDigraph Test
Reflexive∀a: (a,a) ∈ RAll main diagonal = 1Loop at every vertex
Symmetric(a,b) ∈ R → (b,a) ∈ RMatrix = its own transpose (Mᵀ = M)Every edge has a reverse edge
Antisymmetric(a,b)∈R ∧ (b,a)∈R → a=bmᵢⱼ=1 ∧ mⱼᵢ=1 only if i=jNo two-way arrows between different vertices
Transitive(a,b)∈R ∧ (b,c)∈R → (a,c)∈RM² ∧ M ≤ M (complex)If path a→b→c exists, edge a→c exists
Symmetric vs Antisymmetric are NOT opposites. A relation can be both (e.g., the equality relation: a=b) or neither (e.g., {(1,2),(2,1),(1,3)}). Do not assume excluding one implies the other.

Equivalence Relations

Definition

R is an equivalence relation if and only if it is Reflexive AND Symmetric AND Transitive. All three are required.

Classic example: congruence mod m a ≡ b (mod m) iff m | (a−b) Reflexive: a−a=0, m|0 ✓ Symmetric: if m|(a−b) then m|(b−a) ✓ Transitive: if m|(a−b) and m|(b−c) then m|(a−c) = m|(a−b+b−c) ✓

Equivalence Classes

[a]ᴿ = {all b such that (a,b) ∈ R}. Elements in the same class are "equivalent."

  • Equivalence classes are either identical or disjoint. Never partially overlap.
  • They form a partition of the whole set.
  • [a] = [b] if and only if aRb.
Exam favorite: You'll be given a relation and asked if it's an equivalence relation. Check all three properties and show your work for each.

Relation Matrices — How to Read and Write

R on A = {1,2,3}, with R = {(1,1),(1,2),(2,3),(3,1)} Matrix Mᴿ (row = from, col = to): 1 2 3 1 [ 1 1 0 ] 2 [ 0 0 1 ] 3 [ 1 0 0 ] Reflexive? No — (2,2) and (3,3) missing (diagonal not all 1s) Symmetric? No — (1,2)∈R but (2,1)∉R Transitive? Check: (1,2)∈R and (2,3)∈R → need (1,3)∈R — NOT present → Not transitive
Day 7 Counting — Product/Sum Rules, Pigeonhole, P&C Problem-Heavy ▾

📋 Prerequisites

Basic arithmetic. Factorials. Willingness to reason carefully about whether order matters. This topic requires you to read problems extremely carefully — one word changes the entire approach.

The Two Fundamental Rules

Product Rule

When doing task 1 AND task 2: multiply.

n₁ × n₂ × ... × nₖ Example: 4-digit strings with no repeat = 10 × 9 × 8 × 7 = 5040 (Each position: AND the next position)

Sum Rule

When doing task 1 OR task 2 (mutually exclusive): add.

n₁ + n₂ + ... + nₖ Example: 18 math majors OR 325 CS majors = 18 + 325 = 343 (Picking one type OR the other)

Inclusion-Exclusion

|A₁ ∪ A₂| = |A₁| + |A₂| − |A₁ ∩ A₂| // Use when tasks are NOT mutually exclusive // Example: bit strings of length 8 that start with 1 OR end with 00 |start with 1| = 2⁷ = 128 |end with 00| = 2⁶ = 64 |both| = 2⁵ = 32 Total = 128 + 64 − 32 = 160

Permutations vs Combinations — The Key Decision

Permutations — Order Matters

P(n,r) = n! / (n−r)! = n × (n−1) × ... × (n−r+1) Example: 5 presentations from 27 students P(27,5) = 27×26×25×24×23 = 9,687,600 // Use when: ranked list, sequence, arrangement // "First place, second place..." → Permutation

Combinations — Order Does NOT Matter

C(n,r) = n! / (r! × (n−r)!) = P(n,r) / r! Example: 5-card poker hand from 52 cards C(52,5) = 52!/(5!×47!) = 2,598,960 // Use when: selection, group, committee // "Choose 5 students for a team" → Combination
The #1 Counting Error: Using permutations when the problem says order doesn't matter, or vice versa. Ask yourself: would swapping two elements create a "different" outcome? If yes → Permutation. If no → Combination.

Useful Identity

C(n,r) = C(n, n−r) // Choosing 5 to include = choosing 47 to exclude // C(52,5) = C(52,47). Obvious but useful for simplification.

Pigeonhole Principle

Basic Pigeonhole

If k+1 objects go into k boxes, at least one box has 2+ objects.

367 people → some two share a birthday (366 possible birthdays = 366 "boxes") 27 words → two start with same letter (26 letters = 26 "boxes")

Generalized Pigeonhole

If N objects go into k boxes, at least one box has ⌈N/k⌉ objects.

Ensure 6 students with same grade (A/B/C/D/F): k = 5 boxes (grades) ⌈N/5⌉ ≥ 6 → N/5 > 5 → N > 25 → N = 26 // Formula: set ⌈N/k⌉ = target, solve for N

Circular Permutations

Seating n people at a circular table: (n−1)! arrangements ← fix one person, arrange the rest Example: 6 people at circular table = (6−1)! = 5! = 120 Alternative: 6!/6 = 120 (total arrangements / rotations)
Day 8 Mathematical Induction — The Hardest Topic Hardest Day ▾

📋 Prerequisites — Critical

You need summation formulas from Day 4 cold. Induction proofs for summations require you to manipulate the summation algebraically during the inductive step. You also need to understand the difference between what you are given (inductive hypothesis) and what you must prove (inductive step). Without Day 4, this day will be impossible.

How Induction Actually Works (Conceptually)
You prove a chain: P(1) is true. If P(k) is true, then P(k+1) is true. This is like dominos — knock down the first one, and since each knocks the next, they all fall. The exam will test if you understand WHY each step is required, not just that you memorized the template.

The Template — Never Deviate From This Format

Goal: Prove P(n) for all positive integers n 1. Base Case: Show P(1) is true. ← plug n=1 into both sides, verify LS = RS 2. Inductive Hypothesis (I.H.): Assume P(k) is true for some arbitrary positive integer k. ← write out P(k) explicitly MUST LABEL THIS CLEARLY — lose marks if missing 3. Inductive Step: Using the I.H., prove P(k+1) is true. ← this is the hard part Strategy: start from the LHS of P(k+1), split off the last term, substitute I.H., simplify to RHS of P(k+1). Conclusion: By mathematical induction, P(n) is true for all positive integers n. ∎

Full Worked Example — Do This Yourself After Reading

Prove: ∑(i=1 to n) i = n(n+1)/2 for all n ≥ 1 Base Case (n=1): LHS = ∑(i=1 to 1) i = 1 RHS = 1(1+1)/2 = 1 ✓ Inductive Hypothesis: Assume ∑(i=1 to k) i = k(k+1)/2 for some k ≥ 1. Inductive Step — Prove P(k+1): Need to show: ∑(i=1 to k+1) i = (k+1)(k+2)/2 LHS = ∑(i=1 to k+1) i = [∑(i=1 to k) i] + (k+1) ← "the thing": split off last term = k(k+1)/2 + (k+1) ← substitute I.H. = k(k+1)/2 + 2(k+1)/2 ← common denominator = (k+1)(k+2)/2 ← factor out (k+1) = RHS of P(k+1) ✓ By mathematical induction, the formula holds for all n ≥ 1. ∎

The Critical Technique — "The Thing"

∑(i=1 to k+1) f(i) = ∑(i=1 to k) f(i) + f(k+1) = [I.H. gives you this] + f(k+1) This is how you get the I.H. into the inductive step every single time. Peel off the last term. See the k-sum. Apply I.H. Simplify to target.
Biggest Induction Error: Students write "assume P(k+1)" in the hypothesis instead of "assume P(k)". You assume P(k) and PROVE P(k+1). These are different.
Second Biggest Error: In the inductive step, going in the wrong direction. Start from the LEFT SIDE of what you need to prove (P(k+1)), and show it equals the RIGHT SIDE. Do not start from the I.H. and hope you reach P(k+1).

Second Practice Example — Odd Integers

Prove: ∑(i=1 to n) (2i−1) = n² for all n ≥ 1 Base Case (n=1): LHS = 2(1)−1 = 1. RHS = 1² = 1. ✓ Inductive Hypothesis: Assume ∑(i=1 to k) (2i−1) = k² Inductive Step — Prove: ∑(i=1 to k+1) (2i−1) = (k+1)² LHS = [∑(i=1 to k) (2i−1)] + (2(k+1)−1) = k² + (2k+1) ← I.H. + peel last term = (k+1)² ✓ By induction, holds for all n ≥ 1. ∎

How to Use This Day
